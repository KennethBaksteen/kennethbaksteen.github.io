Course Year Backend Frontend Team
Programming Visual User Environments 2nd Bachelor C# C# Solo

IoToilet

Course Year Backend Frontend Team
Project 2 2nd Bachelor Java HTML & CSS 3 people

During a project week about Internet of Things, I developed IoToilet with two other students. This project was developed using several sensors and was used to track toilet visitations on a public toilet. When someone entered the toilet, a song started playing and a next or previous song could be selected by the toilet user. Moreover the duration of a toilet visitation was tracked and saved. At the end of a day, all the data was showed in a web application so that conclusions could be made. This data could then for example be used to efficiently send the cleaning crew to only used toilets.

Android Breating App

Course Year Backend Frontend Team
Graduation Project 3rd Bachelor React Native React Native 3 people

For my graduation project in my bachelor, I worked together with 2 other students on an Android application. The purpose of this application was to help people who experience a lot of stress or breathing problems. Blockr

Course Year Backend Frontend Team
Software Design Switching Program Java Java 4 people

For this project, we had to create a game which looks a lot like Scratch using only Java. For the user interface we were only allowed to use Java and a given CanvasWindow. The total duration of the project was divided into 3 iterations and during each iteration we had to extend the game with new features. During the last iteration we created a user interface where a specific game and a specific way of controlling the game could be chosen (keyboard or buttons on the interface). We also created a simple version of Minesweeper to test this interface. When developing the game we had to use refactorings and several design patterns like for example Strategy, Factory, Facade, Composite, Decorator and many more. We also had to apply GRASP which stands for Generalized Responsibility Assignment Software Patterns/Principles containing for example low coupling and high cohesion programming. The project consisted of more than just the program since we also had to write for example our own use cases, sequence diagrams and UML diagrams. Moreover we had to write our own test cases to achieve a very good code coverage.

Toward accurate robotic simulation by calibration
A research about the accuracy and the calibration of the TurtleBot3 simulation by Gazebo
Course Year Backend Frontend Team
Master's Thesis Master ROS & Python Gazebo Solo

Nowadays more and more robots are released on the market and we become more and more dependent of them. Think about for example automated lawn mowers and vacuum cleaners. All these robots are developed and tested before they are released and this can be done by either using a robot's simulation or the real robot itself. Using a simulation offers several advantages. It is for example cheaper, safer and faster, but there is also one important question that arises. How good do the results of running a program on a simulation represent the results of running the same program on the real robot? In the first part of my thesis I investigated this question for a specific example of a robot, the TurtleBot3 and the simulation of this robot provided by Gazebo. I created several programs and I ran these programs on both the simulation and the real robot. After looking at the results, I was able to calculate a deviation of 1.15%. In the second part of my thesis, I investigated a way to improve the deviation by calibrating the simulation. I was able to decrease the deviation to 0.83%.
